eact 19 now lets you preload resources directly from React instead of relying only on <link> tags in your HTML. Let's check how 👇 

Until now, we've mostly relied on HTML tags like <link rel="preload">, <link rel="preconnect">, or framework-specific optimizations to load critical resources early.

But, with React 19, resource loading can now become part of your component logic instead of living entirely in the <head>.

Here are the new APIs:

- prefetchDNS(): It resolve DNS early for domains you'll likely connect to.

- preconnect(): It establish TCP/TLS connections before the actual request is made.

- preload(): It fetches fonts, stylesheets, images, scripts, or other critical resources in advance.

- preloadModule(): Start downloading an ES module before it's needed.

- preinit() / preinitModule(): Download and initialize scripts/modules early so they're ready when React needs them.

And I find all of them quite interesting because:

- Preloading becomes dynamic instead of static.
Rather than declaring every possible resource in the document <head>, you can preload resources based on user actions, fetched data, or navigation paths.

- Resource loading moves closer to the component that actually needs it.
The logic and the optimization live together, making the code easier to understand and maintain.

- Cleaner and less error-prone.
You no longer have to manage a growing list of <link> tags manually across different pages.
